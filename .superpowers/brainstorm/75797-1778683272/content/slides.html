§2 — Problem Statement
3 / 13
Problem Statement
Communities see the changes every day — the water colour shifting, fish disappearing, farms encroaching. That knowledge has no formal channel into decisions.
📡
Connectivity
Many sites have no mobile data. The system needs to work on a basic voice network alone.
📱
Device access
Not everyone has a smartphone. Pollution reporting must work on any basic phone.
📚
Digital literacy
The interface must guide users through every step. No reading required. No manual.
§3 — Solution Overview · 1 of 2
4 / 13
Guiding Principles
🔓
Open source first
Everything we build is open source. KoboToolbox, FastAPI, PostgreSQL, Next.js — no proprietary tools. No lock-in.
GEE free tier for non-commercial use
📡
Works without data
USSD runs on voice only. KoboCollect saves offline and syncs when there's a connection. No data plan needed.
Designed for the most remote sites
🤝
Data belongs to communities
The data belongs to NBD and the communities who collected it.
Formal Data Processing Agreement
🔗
One ID connects everything
Every site has a fixed ID — e.g. NBD-MARA-001. It appears in every record, every export, every API call. Any dataset can join on that key alone.

§7 — Security, Privacy & Data Governance
10 / 13
Security & Governance
Threats & mitigations
Watcher identity
Reports show basin-level counts only. No submission can be traced to a phone number.
Admin layer
Role-based access control.
Infrastructure
No secrets in code. TLS everywhere. Encryption at rest. Automated patching.
Authentication
Admin interface  SSO — Google / Microsoft
Portal partners  Email + password
Public portal  No login required
No self-registration — Admin creates all accounts via invite
Data ownership
Owners: NBD + communities
The data belongs to NBD and the communities who collected it.
Processor: Akvo
Akvo runs the platform for NBD under a formal Data Processing Agreement. We have no rights over the data beyond operating the platform.
